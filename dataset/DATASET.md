## فرمت فایل

- کدگذاری: UTF-8
- قالب: JSON Lines (`.jsonl`)
- هر خط: دقیقاً یک شیء JSON و یک مقاله
- تعداد فیلد: 9 فیلد با ترتیب ثابت
- خط پایانی: هر رکورد، از جمله آخرین رکورد، با newline خاتمه می‌یابد

ترتیب استاندارد فیلدها:

```text
paper_id
title
abstract
title_abstract
authors
categories
primary_category
year
update_date
```

## قرارداد فیلدها

| فیلد | نوع JSON | الزام و معنا |
|---|---|---|
| `paper_id` | string | شناسهٔ یکتا و غیرخالی مقاله در arXiv؛ فرمت جدید یا قدیمی |
| `title` | string | عنوان پاک‌شده، غیرخالی و قابل نمایش |
| `abstract` | string | چکیدهٔ پاک‌شده، غیرخالی و قابل نمایش |
| `title_abstract` | string | دقیقاً `title + " " + abstract` و همیشه بازسازی‌شده |
| `authors` | string | متن نویسندگان؛ به آرایه شکسته نشده است |
| `categories` | array[string] | حداقل یک دسته‌بندی معتبر، بدون تکرار و با حفظ ترتیب اولیه |
| `primary_category` | string | دسته‌بندی اصلی که حتماً عضو `categories` است |
| `year` | integer | سال استخراج‌شده از `update_date` |
| `update_date` | string | تاریخ معتبر با قالب دقیق `YYYY-MM-DD` |

دو قالب معتبر `paper_id` در خروجی وجود دارد:

```text
# فرمت جدید - 48,997 رکورد
2501.01046
2103.07050

# فرمت قدیمی - 1,003 رکورد
gr-qc/0412088
cmp-lg/9410003
```

## قواعد پاک‌سازی اعمال‌شده

1. فایل به‌صورت خط‌به‌خط خوانده شد و هر خط به‌طور مستقل به JSON تبدیل شد.
2. رکوردهای غیرشیء، JSON نامعتبر یا فاقد فیلدهای ضروری قابل حذف هستند.
3. رکورد دارای `title` یا `abstract` غیررشته‌ای یا خالی پس از پاک‌سازی قابل حذف است.
4. متن‌ها با Unicode NFC نرمال شدند.
5. نویسه‌های کنترلی و نامرئی با دسته‌های Unicode `Cc` و `Cf` حذف شدند؛ tab و newline به فاصله تبدیل شدند.
6. فاصله‌های متوالی به یک فاصله تبدیل و فاصلهٔ ابتدا و انتهای متن حذف شد.
7. علائم نگارشی، URL، فرمول‌های علمی، بک‌اسلش و LaTeX حفظ شدند.
8. `paper_id` به‌صورت رشته حفظ و فقط بر اساس شناسه رفع تکرار شد؛ عنوان تکراری باعث حذف رکورد نشد.
9. `categories` به آرایه‌ای معتبر، بدون تکرار و با ترتیب پایدار تبدیل شد.
10. `primary_category` در صورت ناسازگاری با اولین دسته‌بندی معتبر قابل تعمیر است.
11. `year` برای تمام رکوردها از `update_date` استخراج شد.
12. `title_abstract` برای تمام رکوردها از نسخهٔ پاک‌شدهٔ عنوان و چکیده بازسازی شد.

کارهای زیر عمداً انجام نشده‌اند:

- تبدیل دائمی متن به حروف کوچک
- حذف stopword
- stemming یا lemmatization
- حذف علائم نگارشی، URL یا LaTeX
- شکستن سادهٔ نویسندگان بر اساس ویرگول
- حذف مقاله فقط به‌دلیل تکراری‌بودن عنوان

این پردازش‌های زبانی، در صورت نیاز، باید در لایهٔ تحلیل متن انجام شوند و نباید نسخهٔ قابل نمایش داده را تخریب کنند.
